Question-1: What are the different ways to select an element in the DOM?

We can use getElementById, getElementsByClassName, getElementsByTagName, querySelector, or querySelectorAll to select elements by ID, class, tag, or CSS selector.

Question-2: What is the difference between innerHTML, innerText, and textContent?

innerHTML This lets us see and edit everything inside an element, including the HTML tags, spaces, and formatting. It’s like peeking at the actual code behind that section of the webpage.
innerText shows only the text that’s visible on the screen, just as a user would see it. It skips anything hidden and pays attention to styling, so it’s great for grabbing just the text that’s displayed.
textContent gives us all the text inside an element, even if it’s hidden, but without any of the HTML tags or formatting. It’s like reading the plain, unformatted text, including parts that aren’t visible to the user.

Question-3: How do you add an event listener in JavaScript?

We can use addEventListener to add an event listener in JavaScript. For example, we can use document.querySelector('button') to get the button. The listener should then be attached like this, button.addEventListener('click', () => alert('Clicked!')); This executes the code inside the function after the click event.

Question-4: What is the purpose of the 'this' keyword in JavaScript?

In JavaScript, "this" denotes the object that is executing the code. The way in which a function is called on changes its meaning. This indicates the object to which the methods belong.

Question-5: How do you create a promise in JavaScript?

We can create a promise using the Promise constructor in JavaScript. This constructor takes a function that has two parameters: resolve and reject. Within this function, we can write our asynchronous code. If the operation completes successfully, we call resolve(value) to indicate success. If there’s an error, we call reject(error) to indicate failure. This approach helps us manage the results of asynchronous tasks clearly and effectively.
